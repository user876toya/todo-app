# Development Flow

## 基本方針

このプロジェクトは、学習と個人開発を兼ねた Todo アプリです。  
AI に実装を丸投げせず、自分で考えて実装し、AI は設計整理・補助・レビューに使います。

## 開発手順

1. ChatGPT で設計整理
2. Issue 作成
3. ブランチ作成
4. Copilot に実装手順・設計思想・実装順序を確認
5. 自分で実装
6. Codex にレビュー依頼
7. 自分で修正
8. PR 作成
9. Codex で最終確認
10. main に merge

## AI の使い分け

### ChatGPT

- 要件整理
- 設計整理
- REST 設計
- Issue 内容の整理
- PR 文の下書き
- 学習内容の言語化

### Copilot

- 実装前の手順確認
- 実装中の補完
- 小さなコード改善
- 命名の相談
- Controller / Service / Repository の責務確認

### Codex

- 差分レビュー
- PR レビュー
- バグになりそうな箇所の指摘
- 責務分割の確認
- REST 命名の確認
- 最終レビュー

## Codex レビュー依頼テンプレート

```txt
〇〇機能を実装しました。

レビューお願いします。

- Controller はリクエスト受付に寄せる
- Service にビジネスロジックを寄せる
- REST を意識した URL 命名にする
- 1 Issue = 1 PR を意識する
